# UNAVAILABLE AND BOUNDARIES

- Proprietary product binaries and closed implementation source are not public knowledge sources and are not represented as code.
- Installed applications are not required for KB use. They are required only for live execution, compiler classpaths drawn from an installation, license-gated behavior, and runtime verification.
- Non-text Confluence images and media are represented by official attachment metadata, parent-page identity, URL, media type, and reported size. They are not embedded because the corpus is Markdown-only. Text, source, archive, Office/XML, SVG, and PDF attachments are extracted when publicly downloadable and within the acquisition limit.
- Formal PDFs are represented through independent semantic and layout-preserving extraction. Complex figures may require the source URL; official machine-readable models, libraries, XMI, and examples are also present in the release repository corpus.
- Some versioned Javadoc indexes publish links that return HTTP 404. Each version catalog records those failures. Missing pages are never fabricated.
- The corpus is a dated capture. Future hotfixes and releases are outside the capture until refreshed.
- Runtime-specific project content, private Teamwork Cloud data, customer profiles, licenses, and organization-specific conventions cannot be inferred from public documentation.
