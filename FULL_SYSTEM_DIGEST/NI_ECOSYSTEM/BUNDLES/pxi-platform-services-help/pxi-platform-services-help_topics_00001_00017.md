### Frequently Asked Questions

**Can I route PXI_Trig0 in Bus 1 to PXI_Trig0 in Bus 2 without also routing it to PXI_Trig0 in Bus 3?**

No. When a trigger is routed from a bus, it is routed from the source bus to all other buses in the chassis.

**Can I route PXI_Trig0 in Bus 1 to PXI_Trig1 in Bus 2?**

No. Each route connects the trigger on the source bus to the same trigger on each other bus in the chassis.

**Can I route triggers between two chassis?**

This depends on the modules present in your chassis. Some modules can source triggers from the chassis backplane and drive them to a module in another chassis over a cabled connection. Consult the manuals for your PXI/PXI Express modules for additional details.

### Programmatically Routing and Reserving Triggers

#### Programmatically Routing and Reserving Triggers in VISA

Programmatically routing and reserving triggers is possible using NI-VISA 3.0 and later on a per-module basis. After opening a VISA session to a PXI INSTR resource, you can reserve a trigger line on the trigger bus connected to the device. Additionally, trigger routing and reservation is available through a PXI BACKPLANE resource, available in NI-VISA 3.1 and later. For more information, refer to your NI-VISA documentation.

#### Programmatically Routing and Reserving Triggers in NI-DAQmx

NI-DAQmx supports programmatically routing and reserving triggers on a per-instrument basis. Refer to *Triggering* in the *NI-DAQmx Help* for complete documentation.

### PXI System Settings

**PXI System Settings** is located under **My System** in MAX.

#### PXI Resource Manager

The Active PXI Resource Manager is responsible for creating and updating system description files that describe any PXI hardware in a system.

The PXI Systems Alliance (PXISA) manages the specifications for PXI which determine how a Resource Manager is allowed to behave in a system. While a system can contain multiple PXI Resource Managers, only one can be designated as the Active PXI Resource Manager.

In MAX, select **National Instruments** as the Active Resource Manager to make the following changes:

- Set or modify a chassis number
- Set or modify a chassis model

MAX provides the option of selecting the Active Resource Manager from what is available on the system.

##### Selection Mode

- Automatically Selected —The Active Resource Manager is automatically selected according to rules set by the PXISA specifications.
- User Selected —The Active Resource Manager is selected by the user.

**Active Resource Manager** (Read-only if Selection Mode is set to Automatic)

- (None Selected) —An Active Resource Manager has not been selected.
- <Resource Manager> —The selected Resource Manager is set to active.

The selector displays all Resource Managers that are detected in the PXISA services tree.

For more information about the PXI Resource Manager, refer to www.pxisa.org.

#### PXI Trigger Manager

A PXI Trigger Manager is responsible for:

- Reservation functionality for the trigger bus lines in the chassis.
- Routing functionality for the trigger bus lines in the chassis.

### Trigger Routing and Reservation

#### What Are PXI Triggers?

PXI triggers are signals for synchronizing events between devices in a chassis. Devices in the chassis assert or deassert these signals, and they are passed to other devices through trigger lines on the chassis backplane. Each trigger bus on a chassis backplane contains eight PXI trigger lines (PXI_Trig0 through PXI_Trig7). Each slot in a PXI chassis is connected to a single trigger bus and can transmit and receive signals over the trigger lines from any other device on the same bus. Most chassis contain only a single trigger bus, but some chassis, such as the PXIe-1095, have multiple trigger buses.

#### Chassis with a Single Trigger Bus

In chassis containing only a single bus, all slots share the same trigger bus and can communicate to each other using this bus. For example, the PXIe-1082 chassis has one trigger bus, and all slots (1–8) reside on this trigger bus. In this situation, a device in any slot can use the backplane to send triggers to a device in any other slot.

#### Chassis with Multiple Trigger Buses

Without further configuration, chassis with multiple trigger buses function like three separate single-segment chassis. A device in a slot connected to one trigger bus cannot use the trigger lines to communicate to a device connected to another trigger bus. To see what slots are assigned to each trigger bus, refer to the **Bus Mappings Legend** area of the **Trigger** view for your chassis.

When properly configured, NI software automatically connects bus segments when an NI driver API references modules on different PXI buses.

Trigger route types are *dynamic* (the default), *Away from Bus 1*, *Away from Bus 2*, and *Away from Bus 3*.

### Troubleshooting

#### Why do my VISA instruments have unknown chassis and slot numbers?

You have not properly configured your PXI system. You must identify the PXI chassis model correctly before geographic addressing such as chassis and slot number is enabled. Follow the procedures in [Getting Started](getting_started.html) to configure your PXI system correctly.

#### Why is my PXI LabVIEW Real-Time target disconnected?

MAX cannot find your system on the network. Ensure that the system is powered on and verify that the network equipment is functioning properly.

For more information about LabVIEW Real-Time target states, refer to System States in the MAX Remote Systems Help.
