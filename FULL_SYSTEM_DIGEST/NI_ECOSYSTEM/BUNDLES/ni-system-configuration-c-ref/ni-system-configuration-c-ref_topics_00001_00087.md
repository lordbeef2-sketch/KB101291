### Basic Functions

Use [NISysCfgInitializeSession](nisyscfginitializesession.html) before calling any other system configuration function. Unless specified otherwise, the function selects the local system. Provide a username and password to gain access to a remote system. Pass the session returned from NISysCfgInitializeSession to other NI System Configuration functions to enable them to operate on your specified target system.

Use [NISysCfgFindHardware](nisyscfgfindhardware.html) to find hardware (for example: devices and chassis). Specify what types of hardware you want by using a filter. You must call [NISysCfgNextResource](nisyscfgnextresource.html) to get each item. For more information about filters, refer to [NISysCfgCreateFilter](nisyscfgcreatefilter.html) and [NISysCfgSetFilterProperty](nisyscfgsetfilterproperty.html).

Use [NISysCfgFindSystems](nisyscfgfindsystems.html) to detect targets on the network. You can specify what types of systems you want, and you can specify the output format of each detected system. The output is an enumerator. You must call [NISysCfgNextSystemInfo](nisyscfgnextsysteminfo.html) to get a string representing each detected system. This string can be passed into the NISysCfgInitializeSession function to create a NISysCfgSessionHandle.

Use the [NISysCfgCloseHandle](nisyscfgclosehandle.html) function to close a reference to a previously allocated filter, resource, session, or enumerator.

### Enumeration Methods

Use [NISysCfgCreateComponentsEnum](nisyscfgcreatecomponentsenum.html) and [NISysCfgAddComponentToEnum](nisyscfgaddcomponenttoenum.html) to create a list of software components that can then be installed to a remote system using [NISysCfgInstallUninstallComponents2](nisyscfginstalluninstallcomponents2.html). You also can use [NISysCfgCreateComponentsEnum](nisyscfgcreatecomponentsenum.html) and [NISysCfgAddComponentToEnum](nisyscfgaddcomponenttoenum.html) to create a list of software add-ons that can be installed to a remote system using [NISysCfgInstallSoftwareSet](nisyscfginstallsoftwareset.html).

Use the NISysCfgNext*Info functions listed below to iterate over the contents returned in NISysCfgEnum*Handles, as shown in the following table. When you are finished using a handle, you must call [NISysCfgCloseHandle](nisyscfgclosehandle.html) to release the handle.

| Function | Description |
| --- | --- |
| NISysCfgNextComponentInfo | Returns detailed information about the next component in the supplied componentEnumHandle. |
| NISysCfgNextDependencyInfo | Enumerates details about each dependency between software components as specified in dependencyEnumHandle. This dependency provides details as to why an attempt to install software to an RT target failed. |
| NISysCfgNextExpertInfo | Returns detailed information about the next expert in the supplied expertEnumHandle. |
| NISysCfgNextResource | Returns a handle to the next resource in the supplied enumeration handle. |
| NISysCfgNextSoftwareSet | Returns a handle to the next set in the supplied setEnumHandle. |
| NISysCfgNextSystemInfo | Returns detailed information about the next system in the supplied systemEnumHandle. |

[NISysCfgGetSoftwareSetInfo](nisyscfggetsoftwaresetinfo.html) returns detailed information about a software set.

### Hardware Management

Use [NISysCfgResetHardware](nisyscfgresethardware.html) to restore a hardware device to its previous state. Use [NISysCfgRenameResource](nisyscfgrenameresource.html) to change the name of a hardware device.

Use [NISysCfgSelfCalibrateHardware](nisyscfgselfcalibratehardware.html) to perform self-calibration on a device. Self-calibration adjusts the calibration constants with respect to an onboard reference stored on the device. The new calibration constants are defined with respect to the calibration constants created during an external calibration to ensure that the measurements are traceable to these external standards. The new calibration constants do not affect the constants created during an external calibration because they are stored in a different area of the device memory. You can perform a self-calibration at any time to adjust the device for use in environments other than those in which the device was externally calibrated.

|  | NotesExternal calibration must be performed by a metrology laboratory or another facility that maintains traceable calibration standards. For more information, visit ni.com/calibration. Disconnect or disable any AC input signals before starting self-calibration. AC or varying signals can cause self-calibration to fail or compromise the accuracy of the calibration. Some device types (for example, SCXI modules) do not support self-calibration. Refer to the NI-DAQmx documentation to determine which devices can be self-calibrated. |
| --- | --- |

|  | Tip The analog output channels on E Series devices fluctuate during self-calibration, so you might need to disconnect the AO channels before self-calibration. |
| --- | --- |

Use [NISysCfgSelfTestHardware](nisyscfgselftesthardware.html) to verify that hardware devices are able to perform basic I/O functions. No other tasks should run on the system while executing the self test because the driver may need exclusive access to some device resources. You do not need to disconnect devices from external equipment because the state of I/O lines are maintained throughout the test.

Use [NISysCfgGetResourceProperty](nisyscfggetresourceproperty.html) to get (read) properties for hardware.

Use [NISysCfgSetResourceProperty](nisyscfgsetresourceproperty.html) to set (write) properties for hardware.

Use [NISysCfgGetResourceIndexedProperty](nisyscfggetresourceindexedproperty.html) to get (read) properties for hardware.

Use [NISysCfgSaveResourceChanges](nisyscfgsaveresourcechanges.html) to save property changes on a device.

### Software Management

Use [NISysCfgRestart](nisyscfgrestart.html) to reboot a system. Use [NISysCfgFormat](nisyscfgformat.html) to format a system's hard disk.

Use [NISysCfgCreateSystemImageAsFolder](nisyscfgcreatesystemimageasfolder.html) to save an image folder, a copy of the file contents set up on the primary hard drive of a system. Apply the image to a system using [NISysCfgSetSystemImageFromFolder](nisyscfgsetsystemimagefromfolder.html).

Use [NISysCfgInstallAll](nisyscfginstallall.html), [NISysCfgInstallSoftwareSet](nisyscfginstallsoftwareset.html), [NISysCfgInstallStartup](nisyscfginstallstartup.html), [NISysCfgInstallUninstallComponents2](nisyscfginstalluninstallcomponents2.html), and [NISysCfgUninstallAll](nisyscfguninstallall.html) to add and remove software on a remote target.

Use [NISysCfgGetSystemProperty](nisyscfggetsystemproperty.html) and [NISysCfgSetSystemProperty](nisyscfgsetsystemproperty.html) to get and set system properties. Use [NISysCfgSaveSystemChanges](nisyscfgsavesystemchanges.html) to save the changes to the system.

[NISysCfgGetInstalledSoftwareSet](nisyscfggetinstalledsoftwareset.html) to obtain installed software information from local and remote systems. Use [NISysCfgGetAvailableSoftwareComponents](nisyscfggetavailablesoftwarecomponents.html) and [NISysCfgGetAvailableSoftwareSets](nisyscfggetavailablesoftwaresets.html) to gather available software information from the local system that can be installed on a system.

### Utilities

Use [NISysCfgCreateFilter](nisyscfgcreatefilter.html) to create a hardware filter object that is used to query for specific resources in a system.

Use [NISysCfgGetStatusDescription](nisyscfggetstatusdescription.html) to obtain the description string for a given status code.

Use [NISysCfgImportConfiguration](nisyscfgimportconfiguration.html) to import system configuration data from a file to a specified system. Use [NISysCfgExportConfiguration](nisyscfgexportconfiguration.html) to export system configuration data from a specified system into a file. Measurement & Automation Explorer (MAX)need not be installed to use these functions.

|  | Note NISysCfgImportConfiguration and NISysCfgExportConfiguration replace the deprecated MAX Copy Configuration function. |
| --- | --- |

Use [NISysCfgSetRemoteTimeout](nisyscfgsetremotetimeout.html) to set the timeout for [NISysCfgFindHardware](nisyscfgfindhardware.html), [NISysCfgRenameResource](nisyscfgrenameresource.html), [NISysCfgResetHardware](nisyscfgresethardware.html), [NISysCfgSaveResourceChanges](nisyscfgsaveresourcechanges.html), [NISysCfgSaveSystemChanges](nisyscfgsavesystemchanges.html), [NISysCfgSelfCalibrateHardware](nisyscfgselfcalibratehardware.html), and [NISysCfgSelfTestHardware](nisyscfgselftesthardware.html).

Use [NISysCfgGenerateMAXReport](nisyscfggeneratemaxreport.html) to create a report to document a local or remote system. Reports can be generated in three formats: XML, HTML, and technical support reports.

|  | Note This function is a replacement for the deprecated Measurement & Automation Explorer (MAX) Generate Report function. MAX must be installed to use this function. |
| --- | --- |

Both HTML and XML formats provide readable reports. The XML format makes archiving and comparing reports convenient with a configuration management tool. The technical support format generates a ZIP file containing reports that are useful if you need to contact NI Technical Support. The level of support for this function depends on each device's driver software. This function provides a one-way export only. You cannot import the report into MAX.

Use [NISysCfgFreeDetailedString](nisyscfgfreedetailedstring.html) to free the strings allocated by [NISysCfgSelfTestHardware](nisyscfgselftesthardware.html), [NISysCfgSelfCalibrateHardware](nisyscfgselfcalibratehardware.html), [NISysCfgSaveResourceChanges](nisyscfgsaveresourcechanges.html), [NISysCfgSaveSystemChanges](nisyscfgsavesystemchanges.html), [NISysCfgNextComponentInfo](nisyscfgnextcomponentinfo.html), [NISysCfgGetSoftwareSetInfo](nisyscfggetsoftwaresetinfo.html), [NISysCfgNextDependencyInfo](nisyscfgnextdependencyinfo.html), and [NISysCfgGetStatusDescription](nisyscfggetstatusdescription.html).

Use [NISysCfgTimestampFromValues](nisyscfgtimestampfromvalues.html) and [NISysCfgValuesFromTimestamp](nisyscfgvaluesfromtimestamp.html) to set and get the second and fractional-second units from a timestamp, such as the last calibration.

### NISysCfgAddComponentToEnum

NISysCfgStatus NISysCfgAddComponentToEnum(
 NISysCfgEnumSoftwareComponentHandle componentEnumHandle,
 const char ID[],
 const char version[],
 NISysCfgVersionSelectionMode mode
 );

#### Purpose

Adds a software component to a handle created using [NISysCfgCreateComponentsEnum](nisyscfgcreatecomponentsenum.html).

#### Parameters

| Input |  |  |
| --- | --- | --- |
| Name | Type | Description |
| componentEnumHandle | NISysCfgEnumSoftwareComponentHandle | Specifies a handle that was created using NISysCfgCreateComponentsEnum |
| ID | const char [] | ID of the software component to be added to the enumeration. |
| version | const char [] | Desired version of the software component being added to the enumeration. This field is optional as long as the version selection mode is set to Highest. |
| mode | NISysCfgVersionSelectionMode | Choose whether an exact version or the highest version of the software component specified by ID should be added to the enumeration. |

#### Return Value

| Name | Type | Description |
| --- | --- | --- |
| returnValue | NISysCfgStatus | The error code returned by the function in the event of an error or warning. A value of 0 indicates success. A positive value indicates a warning. A negative value indicates an error. Refer to Status Codes or nisyscfg_errors.h for details regarding error codes. |
