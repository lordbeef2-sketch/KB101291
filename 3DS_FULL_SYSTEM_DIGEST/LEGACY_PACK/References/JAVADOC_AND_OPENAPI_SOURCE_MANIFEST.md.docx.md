# JavaDoc and OpenAPI Source Manifest

Snapshot date: 2026-07-09

## Purpose

This file tells an AI where authoritative Java/API details must come from for Cameo/MagicDraw/CATIA Magic plugin work.

It exists because "give the AI all javadocs" must be handled carefully:

- vendor JavaDocs may be proprietary/licensed

- local installed JavaDocs can differ by version/refresh

- public docs may point to one version while your deployed target is another

- copying entire vendor JavaDocs into an AI pack can create license and staleness problems

The right pattern is: index and route, do not pretend.

## Source priority

1. Local installed JavaDocs for the exact target version

2. Local openapi/examples from the exact target version

3. Official online JavaDocs for the matching or closest version

4. Official Developer Guide pages

5. Existing pack recipes and examples

6. AI inference only after source surfaces are checked

## Known JavaDoc locations

Local installation:

```text

<modeling tool installation directory>\openapi\docs

<modeling tool installation directory>\openapi\examples

<modeling tool installation directory>\lib

<modeling tool installation directory>\plugins\<plugin_name>

```

Public online example:

```text

https://jdocs.nomagic.com/2024x/

```

Public JavaDocs may not cover every installed plugin or every target refresh. Always prefer local installed docs for production code.

## Public online JavaDoc roots by target line

Use these as first-class version-line JavaDoc sources for AI coding lookup. They do not replace local installed JavaDocs for production builds, but they are the correct public roots for fast package/class/method discovery.

```text

2022x: https://jdocs.nomagic.com/2022x/

2024x Refresh3: https://jdocs.nomagic.com/2024xRefresh3/

2026x: https://jdocs.nomagic.com/2026x/

```

Routing rule:

- If user target is 2022x, check the 2022x JavaDocs first.

- If user target is 2024xR3 / 2024x Refresh3, check the 2024xRefresh3 JavaDocs first.

- If user target is 2026x / 2026xR1 and exact R1 JavaDocs are unavailable publicly, check 2026x online JavaDocs and then verify against local installed 2026xR1 JavaDocs before production code.

## API scope rule

Only use API elements marked as OpenAPI.

Do not use:

- InternalAPI

- NotAPI

- deprecated internal-looking calls

- classes only discovered by decompiling jars

- UI/internal implementation classes unless official docs explicitly allow it

- index only after the model-element layer is stable, because diagram APIs are a separate surface from model element APIs.

## AI usage rule

Before producing production plugin code, an AI must say which source it used:

```text

API source checked:

- local JavaDocs:

- online JavaDocs:

- Developer Guide:

- pack recipe:

- unverified inference:

```

If the answer depends on an exact method signature and local JavaDocs are unavailable, the AI must mark the code as draft/pseudocode until the local JavaDocs are checked.

## What not to ingest blindly

Do not dump the entire install tree into the AI library.

Avoid:

- license files

- private connection strings

- auth configs

- local workspace paths with usernames if sensitive

- proprietary jars unless usage is permitted

- internal customer/project models

- TWC tokens, keystores, certificates, passwords
