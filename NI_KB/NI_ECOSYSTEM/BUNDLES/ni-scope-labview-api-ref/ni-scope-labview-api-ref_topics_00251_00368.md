### Clocking:Advanced:Oscillator Phase DAC Value

Gets or sets the binary phase DAC value that controls the delay added to the phase-locked loop (PLL) of the sample clock.

**Valid Values**: [0, 65535]

### Triggering:Trigger Runt:Runt Polarity

Specifies the polarity of pulses that trigger the oscilloscope for runt triggering.

This property determines how the oscilloscope triggers relative to the runt thresholds you set.

When set to **Positive**, the oscilloscope triggers when the following conditions are met:

- The leading edge of a pulse crosses the Runt Low Threshold in a positive direction;
- The trailing edge of the pulse crosses the Runt Low Threshold in a negative direction; and
- No portion of the pulse crosses the Runt High Threshold .

When set to **Negative**, the oscilloscope triggers when the following conditions are met:

- The leading edge of a pulse crosses the Runt High Threshold in a negative direction;
- The trailing edge of the pulse crosses the Runt High Threshold in a positive direction; and
- No portion of the pulse crosses the Runt Low Threshold .

When set to **Either**, the oscilloscope triggers in either case.

Default Value: Positive

### Utility Functions

Use the NI-SCOPE Utility VIs to perform various tasks such as resetting the digitizer and returning the revision numbers of the driver and the instrument firmware.

[IMAGE alt='icon' src='niscope-utility-mnu.png']

- [niScope Reset VI](../../instr-lib/niscope/niscope-llb/niscope-reset-vi.html) Resets the digitizer to its default state.
- [niScope Reset Device VI](../../instr-lib/niscope/niscope-llb/niscope-reset-device-vi.html) Performs a hard reset of the device. Acquisition stops, all routes are released, RTSI and PFI lines are tristated, FPGAs are reset, hardware is configured to its default state, and all session attributes are reset to their default states.
- [niScope Disable VI](../../instr-lib/niscope/niscope-llb/niscope-disable-vi.html) Aborts the current task, opens the data channel relays, and releases all RTSI and PFI lines. Use this VI for a condition where you want to stop the current acquisition and disable the channels.
- [niScope Probe Compensation Signal Start VI](../../instr-lib/niscope/niscope-llb/niscope-probe-compensation-signal-start-vi.html) Generates a 1 kHz square wave signal for probe compensation.
- [niScope Probe Compensation Signal Stop VI](../../instr-lib/niscope/niscope-llb/niscope-probe-compensation-signal-stop-vi.html) Disables the 1 kHz square wave signal for probe compensation.
- [niScope Error Message VI](../../instr-lib/niscope/niscope-llb/niscope-error-message-vi.html) Takes the error code returned by NI-SCOPE VIs and returns the interpretation as a user-readable string.
- [niScope Self Test VI](../../instr-lib/niscope/niscope-llb/niscope-self-test-vi.html) Runs the device self-test routine and returns the test result(s).
- [niScope Revision Query VI](../../instr-lib/niscope/niscope-llb/niscope-revision-query-vi.html) Returns the revision numbers of the instrument driver and instrument firmware.
- [niScope CableSense Signal Start VI](../../instr-lib/niscope/niscope-llb/niscope-cablesense-signal-start-vi.html) Generates the CableSense signal on all channels of an oscilloscope for which the signal is enabled, as configured by the CableSense Mode property.
- [niScope CableSense Signal Stop VI](../../instr-lib/niscope/niscope-llb/niscope-cablesense-signal-stop-vi.html) Disables the CableSense signal on all channels of an oscilloscope for which the signal is enabled.
- [MeasurementLink](../../instr-lib/niscope/niscope-measurementlink-mnu.html) Use the VIs on this palette in conjunction with MeasurementLink Session Management.
- [niScope Is Device Ready VI](../../instr-lib/niscope/niscope-llb/niscope-is-device-ready-vi.html) Returns whether the device is ready to be used to the Device Ready? parameter.
- [niScope Get Session Reference VI](../../instr-lib/niscope/niscope-llb/niscope-get-session-reference-vi.html) Extracts a session that can be passed to NI-TClk VIs. Session References are of generic type, which means that the corresponding wires are blue-green, unlike the wires for regular instrument driver sessions.
- [niScope Get Stream Endpoint Handle VI](../../instr-lib/niscope/niscope-llb/niscope-get-stream-endpoint-handle-vi.html) Returns a writer endpoint that can be used with NI-P2P to configure a peer-to-peer stream with a digitizer endpoint.
- [niScope Get Channel Name (Poly) VI](../../instr-lib/niscope/niscope-llb/niscope-get-channel-name-poly-vi.html) Returns a comma-delimited list of channel names.
